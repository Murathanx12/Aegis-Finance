#### Finding 1: `drift_detector.py:199` — Legacy `np.random` violates reproducibility rule (BUG)

**File:** `backend/services/drift_detector.py:199`  
**Severity:** Medium — correctness + rule violation

The `_ks_test` method uses `np.random.uniform()` (legacy global RNG) to generate synthetic reference samples for the KS test. This violates the project rule "Use `np.random.default_rng(seed)` for reproducibility" and means:
- KS test results are **non-reproducible** across runs
- Results depend on global RNG state, creating hidden coupling between test calls
- Each call to `check_drift()` produces different p-values for the same data

**Fix:** Accept `rng` in `__init__`, store it, and use `self._rng.uniform()` in `_ks_test`. Alternatively, store the actual reference values (not just bin edges) to avoid synthetic sampling entirely — this is both simpler and eliminates the approximation error from reconstructing the distribution.

#### Finding 2: `regime_detector.py` — Pure rule-based, no HMM despite infrastructure existing (ARCHITECTURE GAP)

**File:** `backend/services/regime_detector.py:46-123`  
**Severity:** High — this is the single biggest model quality gap

The regime detector is entirely rule-based (rolling returns + vol thresholds + VIX/risk overlays). The Monte Carlo engine (`monte_carlo.py:111-113,146-148`) already accepts `hmm_state_means`, `hmm_regime_probs`, and `hmm_state_vols` — but **nothing ever provides them**. The `hmmlearn` library is listed as installed in CLAUDE.md but never imported anywhere in the codebase.

This matters because:
- The rule-based detector uses a **252-day rolling window** — it's severely lagging. The 2025 tariff selloff override (lines 82-99) is a bandaid for this structural lag.
- HMM would provide **probabilistic regime assignments** (soft transitions) instead of hard thresholds, directly feeding the MC engine's existing blend parameters.
- The `hmm_drift_blend` (0.15) and `hmm_vol_blend` (0.15) in config are dead parameters — they never activate.

**Fix:** Fit a 3-state Gaussian HMM on rolling returns + vol features in `regime_detector.py`, extract state means/vols/transition matrix, and wire the posterior probabilities into the MC engine. The MC infrastructure is already there.
