# V4 Roadmap — "a quant desk for the average person" (2026-07-16)

Owner intent (Murat): the engine should act like a hedge-fund analyst for a
regular person — find candidate stocks, manage the portfolio, prove itself on
the honest paper lanes, and stay cheap enough to run for the full 24-month
record. Compete on trust (forward record) + accessibility (casual UI), not on
feature count.

Standing constraints (unchanged, non-negotiable):
- Paper-lane write path is sacred; no LLM anywhere near a trade decision
  (every leakage-controlled live test shows LLM traders lose money — see
  `docs/research/BEST_PROJECTS_SYNTHESIS_2026-07-14.md`).
- New signals require pre-registration; six forward clocks is ENOUGH — each
  new one raises the multiple-testing bar for all of them.
- No skill claims before the 24-month bar (lane-flip lesson 2026-07:
  conviction went +5.5% → −5.4% vs mirror −8.4% → +0.8% in five weeks —
  every short-window read is noise).

## LLM module verdict (Murat asked: replace DeepSeek?)

Keep DeepSeek for what it does today (news→structured JSON, brief summaries,
commentary): it is the cheapest capable model for bulk text→data work, the
output contract is validated JSON with template fallback, and the LLM is
deliberately OFF every trade path — so marginal quality there buys nothing
measurable. Two upgrades worth considering, in order:
1. **Task-split**: keep DeepSeek for bulk/structured extraction; route the
   few USER-FACING paragraphs/day (daily brief prose, portfolio commentary)
   to Claude Haiku 4.5 — noticeably better writing, still cents/day at our
   volume (~5-20 calls/day). `llm_analyzer` already has provider plumbing +
   spend guards.
2. **FinBERT stays** for sentiment scoring (now idle-unloaded); it is not an
   LLM cost and switching sentiment to a chat LLM would cost more and
   validate worse (FinBERT vs FPB/FiQA benchmark is a backlog item).
Do NOT: fine-tune (FinGPT LoRA rejected), or use any LLM as a stock picker.
