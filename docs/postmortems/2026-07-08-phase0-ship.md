## The finding that matters: training was NOT byte-deterministic

`test_training_is_deterministic_given_fixed_input` passed on Windows-local and
failed on CI (sklearn 1.9 / scipy 1.18 / numpy 2.5) — twice, surviving the
obvious fix (LightGBM `deterministic=True`). The claim "two trainings produce
byte-identical artifacts," which the provenance sidecar's `model_sha256` story
leans on, was **false on any Linux box with current libs** and nobody knew,
because the test had only ever run where it happened to hold.

**The diagnosis method is the reusable part.** Instead of guessing again, the
test was rewritten component-wise: sha each pickled piece separately (booster
`model_to_string`, LR coefs+pickle, calibrators, scalers, imputers). CI then
proved every semantic component identical while the file sha differed — which
cornered the culprit into the only thing not compared: the sklearn wrapper's
`evals_result_`/`best_score_` floats, computed by threaded metric reductions
that wobble in the last ULP. They're unused at predict time; `save_model` now
strips them. One diagnostic round-trip replaced an unbounded guessing loop.

## Durable lessons

1. **"Deterministic on my machine" is a platform claim, not a property.** A
   determinism/reproducibility test that has only run on one OS+lib stack has
   not established the property. CI's fresh resolve is the real witness.
2. **When a hash mismatch resists the obvious fix, stop guessing and make the
   test name the culprit** — compare components, then dump the first differing
   byte offset. Assertion messages are a remote debugger when you can't shell
   into the failing environment.
3. **Railway deploys are gated on GitHub CI** ("Waiting for CI" in `railway
   status`). A red CI silently blocks every deploy — a push that "didn't land"
   means check Actions first, not Railway.
4. **CI job logs need auth; the token is already on the machine** —
   `git credential fill` recovers it when `gh` isn't installed.
5. The prod `detect_regime` warning was the swallowed-import class again: a
   function that never existed, imported inside a try/except, degrading regime
   context to a permanent None since 2026-07-01. Fix + regression tests that
   pin the import path and assert no swallowed warning.
